## Задача 25

### Схема БД

- **Product(maker, model, type)**
- **PC(code, model, speed, ram, hd, cd, price)**
- **Laptop(code, model, speed, ram, hd, price, screen)**
- **Printer(code, model, color, type, price)**

Таблица `Product` содержит производителя (`maker`), номер модели (`model`) и тип (`'PC'`, `'Laptop'` или `'Printer'`). Номера моделей уникальны для всех производителей и типов.
В `PC` для каждого ПК указаны: уникальный код, модель (внешний ключ к `Product`), скорость процессора (МГц), объем RAM (МБ), размер диска (ГБ), скорость CD (например, `'4x'`) и цена.
`Laptop` аналогична, но вместо `cd` содержит размер экрана (`screen` в дюймах).
`Printer` содержит: код, модель, цветность (`'y'`/`'n'`), тип (`'Laser'`, `'Jet'`, `'Matrix'`) и цену.

### Задание 25 (Serge I: 2003-02-14)

Найдите производителей принтеров, которые производят ПК с наименьшим объемом RAM и с самым быстрым процессором среди всех ПК, имеющих наименьший объем RAM.  
Вывести: `maker`.

## Задача 55

### Схема БД (корабли)

- **Classes(class, type, country, numGuns, bore, displacement)**
- **Ships(name, class, launched)**
- **Battles(name, date)**
- **Outcomes(ship, battle, result)**

Корабли в классах построены по одному проекту. Классу присваивается либо имя первого корабля (головного), либо имя проекта, не совпадающее ни с одним кораблём.

- `Classes`: имя класса, тип (`bb` – линейный корабль, `bc` – боевой крейсер), страна, число орудий, калибр (дюймы), водоизмещение (тонны).
- `Ships`: название корабля, класс, год спуска на воду.
- `Battles`: название и дата битвы.
- `Outcomes`: корабль, битва, результат (`sunk`, `damaged`, `OK`).

Примечания:
1. В `Outcomes` могут быть корабли, отсутствующие в `Ships`.
2. Потопленный корабль в последующих битвах не участвует.

### Задание 55 (Serge I: 2009-10-13)

Для каждого класса определите год, когда был спущен на воду первый корабль этого класса. Если год спуска на воду головного корабля неизвестен, определите минимальный год спуска на воду кораблей этого класса.  
Вывести: `class`, `year`.

### Задание 105 (qwrqwr: 2013-09-11)

Статистики Алиса, Белла, Вика и Галина нумеруют строки у таблицы `Product`.  
Все четверо упорядочили строки по возрастанию названий производителей.

- **Алиса** присваивает новый номер каждой строке, строки одного производителя упорядочивает по номеру модели.
- **Белла** присваивает один и тот же номер всем строкам одного производителя. Нумерация начинается с 1, каждый следующий производитель увеличивает номер на 1.
- **Вика** присваивает каждому следующему производителю тот же номер, какой получила бы первая модель этого производителя у Алисы.
- **Галина** присваивает каждому следующему производителю тот же номер, какой получила бы его последняя модель у Алисы.

Вывести: `maker`, `model`, номера строк, получившиеся у Алисы, Беллы, Вики и Галины соответственно.

## Задача 150

### Схема БД (пункты приёма вторсырья)

**Таблицы с однократной фиксацией в день:**

- `Income_o(point, date, inc)` – первичный ключ `(point, date)`. Приём денег на пункте не чаще раза в день.
- `Outcome_o(point, date, out)` – первичный ключ `(point, date)`. Выдача денег не чаще раза в день.

**Таблицы с возможностью многократной фиксации в день:**

- `Income(code, point, date, inc)` – первичный ключ `code`.
- `Outcome(code, point, date, out)` – первичный ключ `code`.

Значения `date` не содержат времени.

### Задание 150 (Serge I: 2019-07-05)

Для каждого пункта в таблице `Income` (многократная фиксация) определить минимальную (`min_date`) и максимальную (`max_date`) даты поступления средств.  
В упорядоченной по времени последовательности всех записей в таблице `Income` для каждого интервала `[min_date, max_date]` определить по одной строке непосредственно выше (`date1 < min_date`) и непосредственно ниже (`date2 > max_date`). Другими словами, требуется расширить каждый интервал на одну строку сверху и снизу.  
Если искомой строки/строк нет, считать значение `date1`/`date2` неопределённым (`NULL`).  
Вывести: `point`, `date1`, `min_date`, `max_date`, `date2`.
